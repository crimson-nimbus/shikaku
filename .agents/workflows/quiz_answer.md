# ITパスポート 回答判定ワークフロー

このワークフローは、ユーザーが出題された問題に対して「ア」「イ」「ウ」「エ」のいずれかで回答した際に起動します。

## 記録方式（重要）

> **ターミナルコマンド（`run_command`）は使用禁止。**
> 回答記録は以下の方式で行うこと：
> 1. `view_file` で `data/history.json` を読み込む
> 2. `replace_file_content` で新しいレコードを追記する（JSON配列の末尾`]`の前に新レコードを挿入）

## 手順

1. 直前に出題した問題の情報（問題ID、正解、解説）を参照する。

2. ユーザーの回答と正解を照合し、正誤を判定する。

3. 以下のフォーマットで結果を表示する：

### 正解の場合：
```
✅ **正解！**

**答え：{answer}）{choices[answer]}**

📖 **解説**
{explanation}
```

### 不正解の場合：
```
❌ **不正解...**

**あなたの回答：{user_answer}）{choices[user_answer]}**
**正解：{answer}）{choices[answer]}**

📖 **解説**
{explanation}
```

4. `data/history.json` に回答記録を追記する。以下の形式で記録する：

```json
{
  "timestamp": "（現在の日時をISO 8601形式で）",
  "question_id": "（出題した問題のID）",
  "category": "（分野名）",
  "subcategory": "（中分類名）",
  "user_answer": "（ユーザーの回答）",
  "correct_answer": "（正解）",
  "is_correct": true/false
}
```

**具体的な記録手順：**
- `view_file` で `data/history.json` を読み込む
- `replace_file_content` を使い、JSON配列の最後の `]` の直前にカンマ＋新レコードを挿入する
- 配列が空（`[]`）の場合は、`[]` を `[\n  {新レコード}\n]` に置き換える

5. 回答判定と記録が完了したら、現在の成績サマリーを簡潔に表示する：

```
📊 **現在の成績**
総問題数：{total}問 | 正解：{correct}問 | 正答率：{rate}%

続けて出題する場合は「出題して」と送ってください。
```

## 重要な注意事項

- 「ア」「イ」「ウ」「エ」以外の回答には「ア〜エで回答してください」と促す。
- 記録ファイルへの書き込みは必ず行うこと（記録漏れ不可）。
- history.json が存在しない場合は `write_to_file` で新規作成する。
- **ターミナル（`run_command`）は絶対に使用しないこと。**
